Clase #1 Faces del desarrollo de software. 1) Analisis. - Problematica. - Levantamiento de requicitos. -Funcionales. -No funcionales. - Entrevista. -Usuario final. -Informe de requicitos. -Este se hace con las normas APPA o las normas ICONTEC. 2) Fase de diseño. Se hace un diagrama de estructura y se definen los roles para comenzar el desarrollo del producto. -Diagramas o mapas mentales. -UML. -Diagrama de clases. -Diagrama de paquetes. -Diagrama de despliegue. -Diagrama componbentes. -Diagrama casos de uso. -Diagrama de antividades. -Modelo relacional. 3) Desarrollo. -Grupos de desarrollo. -Comunicadores. -Psicologos. -Desarrollador. -Front. -Back. -Full Stack. 4) Despliegue o Inplementacion. Se simula un servidor temporal y se hacen pruebas que pasen las pruebas de calidad ( QA y Tester). Metodologias para la programacion. Metodologia tredicional y metodologia agil. **Metodologia tradicional. Cada face se trebaja independiente y es concecuencia de la siguiente. Entre mayor sea el tiempo, mas perdida tecnica se tiene que asumir por el uso de la metodologia tradicional en cascada. -Analisis. -Diseño. -Desarrollo. -Implementacion. **Metodologia agil. En la metodologia agil, se tienen muy en cuenta el cliente, para completar el producto satisfaciendo por completo las necesidades de este. -Analisis. -Diseño. -Desarrollo. -Implementacion.
